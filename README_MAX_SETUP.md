# MAX bot without Watbot

Этот проект заменяет Watbot на обычный Python-код.

Схема теперь такая:

```text
MAX -> Railway -> FastAPI -> PPTX + Excel -> ZIP -> ссылка пользователю
```

## Что есть в коде

- `main.py` - весь backend и логика MAX-бота.
- `requirements.txt` - библиотеки, которые Railway установит сам.
- `Procfile` - команда запуска для Railway.

## Какие переменные нужны в Railway

Открой Railway -> твой проект -> вкладка `Variables` и добавь:

```text
BOT_TOKEN=токен_бота_из_MAX
BASE_URL=https://адрес-твоего-проекта.up.railway.app
WEBHOOK_SECRET=любая_строка_без_пробелов_например_mysecret12345
```

`WEBHOOK_SECRET` можно придумать самому. Главное: только английские буквы, цифры, `_` или `-`, длина от 5 символов.

## Как подключить MAX к Railway

После деплоя открой в браузере:

```text
https://адрес-твоего-проекта.up.railway.app/setup_max_webhook
```

Если всё хорошо, MAX ответит примерно:

```text
MAX response 200:
{"success":true}
```

## Как пользоваться ботом

1. Открой бота в MAX.
2. Нажми Start или напиши `/start`.
3. Нажми `Загрузить шаблон`.
4. Отправь `.pptx`.
5. Нажми `Создать грамоты`.
6. Отправь `.xlsx`.
7. Бот пришлёт ссылку на ZIP.

## Как должен выглядеть PPTX

В тексте презентации должны быть слова, которые совпадают с колонками Excel.

Пример в PPTX:

```text
Имя
Класс
Дата
```

Пример колонок Excel:

```text
Имя | Класс | Дата
Иван | 5А | 12.05.2026
Анна | 6Б | 12.05.2026
```

Каждая строка Excel станет отдельной грамотой.

## Важно про файлы на Railway

Сейчас файлы хранятся прямо на диске Railway:

- `templates/`
- `excel/`
- `output/`
- `state/`

Для первого запуска этого достаточно. Для настоящего SaaS лучше потом перенести хранение в базу данных и S3-совместимое хранилище, потому что диск Railway может очищаться при redeploy.
